# Track 9: Next.js Fundamentals (App Router)

## Overview
This track builds a precise mental model of the Next.js App Router: file-system-based routing with nested layouts and dynamic segments, the Server Component / Client Component boundary and how to compose across it, and server-side data fetching including Next.js's `fetch` caching semantics. You are expected to write 100% of the code yourself — the exercise files contain only small illustrative snippets to critique, complete, or fix, never a working solution.

## Before You Start
Before exercise 1, scaffold your own project with `npx create-next-app@latest` (choose App Router, TypeScript optional) inside `workspace/track-9-nextjs-fundamentals/`. Do this yourself — do not ask AI to configure it for you; read the CLI prompts and understand each option before pressing enter.
